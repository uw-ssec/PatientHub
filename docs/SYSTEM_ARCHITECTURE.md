# PatientHub System Architecture

> A unified framework for simulating and evaluating mental health therapy sessions using AI

---

## ğŸ¯ System Goals & Problem Statement

### The Problem

Training and evaluating AI therapists is challenging because:

1. **Scarcity of Real Data**: Access to real therapy sessions is limited due to privacy concerns
2. **Diversity of Patients**: Mental health conditions manifest differently across individuals
3. **Evaluation Complexity**: Therapeutic effectiveness is multi-dimensional and context-dependent
4. **Reproducibility**: Research methods are fragmented across papers with inconsistent implementations

### PatientHub's Solution

PatientHub provides a **unified simulation and evaluation framework** that:

- **Simulates** realistic therapy sessions between AI clients and therapists
- **Evaluates** therapeutic performance using structured LLM-based judges
- **Standardizes** 12 different client simulation methods from published research
- **Enables** reproducible experiments for mental health AI research

## ğŸ“Š System Metrics & Considerations

### Performance Characteristics

| Operation | Typical Latency | Notes |
|-----------|----------------|-------|
| Single turn generation | 1-3 seconds | Depends on model and prompt length |
| Full session (20 turns) | 30-60 seconds | Can be parallelized |
| Session evaluation | 5-15 seconds | Depends on granularity |
| Character generation | 10-30 seconds | One-time operation |

### Scalability Considerations

- **Stateless agents**: Each agent can be instantiated independently
- **Session parallelization**: Multiple sessions can run concurrently
- **Model flexibility**: Switch between fast/cheap and slow/expensive models
- **Data persistence**: JSON-based storage enables easy migration

### Security & Privacy

- API keys stored in `.env` (not committed)
- No real patient data in repository
- Synthetic character profiles only
- Session logs contain simulated data only

## Summary

PatientHub is a comprehensive framework that bridges the gap between mental health research and AI development. By providing:

1. **Standardized client simulations** from 12 published research methods
2. **Flexible evaluation pipelines** with multiple assessment paradigms  
3. **Extensible architecture** for adding new agents and evaluators
4. **Research reproducibility** through consistent interfaces and configurations

The system enables researchers and developers to:
- Train and evaluate AI therapists without real patient data
- Compare different therapeutic approaches systematically
- Generate diverse, realistic therapy scenarios
- Build evidence-based mental health AI applications
